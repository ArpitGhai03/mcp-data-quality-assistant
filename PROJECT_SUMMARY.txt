FILE DETAILS
================================================================================

1. db_setup/create_db.py
   Purpose: Initialize production and staging databases
   Features:
   - Creates orders table with 5 fields
   - Generates 100 production orders using Faker
   - Creates staging data with simulated quality issues
   - 9% data loss (missing rows)
   - 8% data modifications (amount field variations)
   Status: ✅ Working perfectly
   Output: data/prod.db (100 records), data/staging.db (91 records)

2. src/tools.py
   Purpose: Core analysis tools (MCP-compatible)
   Features:
   - get_missing_rows() → Returns 9 missing records
   - get_mismatched_rows() → Returns 8 mismatched records
   - get_quality_score() → Returns 83% quality score
   - run_full_comparison() → Complete comparison report
   - export_report() → Saves to JSON
   Status: ✅ All tools functional
   Dependencies: sqlite3, datetime, json

3. src/agent.py
   Purpose: Ollama-powered AI agent with reasoning loop
   Features:
   - Connects to local Ollama (llama3 model)
   - Natural language query processing
   - Automatic tool selection
   - Multi-iteration reasoning loop (max 3 iterations)
   - JSON-formatted output
   Status: ✅ Tested and working
   Dependencies: ollama (Client)
   Usage: python agent.py "Your query here"
   Example Queries:
   - "What is the data quality score?"
   - "Show me the missing rows"
   - "Generate a full comparison report"

4. src/mcp_server.py
   Purpose: MCP Protocol server for AI assistant integration
   Features:
   - Exposes all tools as MCP endpoints
   - Implements JSON-RPC protocol
   - Proper async/await handling
   - Logging and error handling
   - Ready for Claude Desktop
   Status: ✅ Fully operational
   Dependencies: mcp, asyncio
   Usage: python mcp_server.py
   
5. src/compare.py
   Purpose: Standalone CLI report generator
   Features:
   - Formatted table output
   - Detailed missing/modified row reports
   - Quality metrics display
   - No AI dependency
   Status: ✅ Working
   Dependencies: tabulate
   Usage: python compare.py

6. src/test_mcp_ollama.py
   Purpose: Integration test for Ollama + MCP tools
   Features:
   - Tests 3 different queries
   - Validates tool execution
   - Shows Ollama recommendations
   - Displays results in formatted JSON
   Status: ✅ All tests passed
   Usage: python test_mcp_ollama.py

7. src/dashboard.py
   Purpose: Interactive Streamlit web dashboard
   Features:
   - 3-page web interface:
     * Page 1 (Overview): Quality score gauge, status badge, metrics, charts
     * Page 2 (Detailed): Missing/mismatched rows tables, side-by-side comparison
     * Page 3 (AI Assistant): Natural language queries with tool execution
   - Real-time metrics and KPIs
   - Interactive Plotly charts (gauge, pie, bar)
   - Responsive DataFrames
   - Ollama integration for explanations
   - Quick query buttons
   - System status monitoring
   - Export report functionality
   Status: ✅ Live and fully operational
   Dependencies: streamlit, plotly, pandas, ollama
   Usage: streamlit run dashboard.py
   Access: http://localhost:8501

ARCHITECTURE
================================================================================

WORKFLOW DIAGRAM:

  User Query
      ↓
  Ollama (llama3 model)
      ↓
  Tool Recommendation
      ↓
  Execute Tool from tools.py
      ↓
  Database Query (SQLite)
      ↓
  Data Analysis
      ↓
  Return Results (JSON)
      ↓
  Format & Display


MCP INTEGRATION FLOW:

  Claude Desktop (or any MCP client)
      ↓
  MCP Protocol (stdio)
      ↓
  mcp_server.py
      ↓
  Tool Handlers
      ↓
  Execute tools.py functions
      ↓
  Return Results
      ↓
  Display in Client

TROUBLESHOOTING
================================================================================

Issue: "ModuleNotFoundError: No module named 'faker'"
Solution: pip install faker

Issue: "Ollama connection refused"
Solution: Start Ollama with: ollama serve

Issue: "MCP server timeout"
Solution: Ensure python venv is activated

Issue: "Database not found"
Solution: Run: python db_setup/create_db.py

Issue: "Port already in use"
Solution: Change port in configuration or stop conflicting process
